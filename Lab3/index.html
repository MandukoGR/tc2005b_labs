Laboratorio 1

Introducción a las aplicaciones web, HTML5 y ciclo de vida de los sistemas de información

Preguntas:

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?
  2. ¿Cuáles son las partes de un URL?
  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
  10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
  11. ¿Qué componentes de estructura y estilo tiene una tabla?
  12. ¿Cuáles son los principales controles de una forma HTML5?
  13. ¿Qué tanto soporte HTML5 tiene el navegador que utilizas?
  14. ¿Cuál es el ciclo de vida de los sistemas de información?
  15. ¿Cuál es el ciclo de desarrollo de sistemas de información?

Respuestas:

  1. Internet es una red de computadoras conectadas entre sí, mientras que la World Wide Web es una colección de páginas que se asienta en esa red (BBC News, 2019).
  2. Un URL consta principalmente de cuatro partes: Protocolo de comunicación, dominio principal, ruta de acceso y parámetros
  3. Estos métodos son conocidos como métodos de petición y su función es indicar la acción que se desea realizar para un recurso determinado.
  4. Puede utilizarse GET o POST, dependiendo de como esté programada la página. La diferencia radica en como son enviados los datos. Si se utiliza GET los datos se envían mediante la URL y con POST mediante el protocolo de comunicación.
  5. Se utiliza el método GET para obtener recursos de un servidor.
  6. Que todo funciona correctamente.
  7. Si el usuario ingresó una ruta incorrecta no es responsabilidad del ingeniero, pero si la ruta ha sido generada automaticamente sí lo es.
  8. Sí, porque es un error del servidor y por lo tanto es responsabilidad del ingeniero corregirlo.
  9. Que ya está en desuso. Algunos elementos de HTML 4 que ya están desaprobados en HTML 5 son: applet, dir, isindex, menu.
  10. Con HTML 5 no es necesario utilizar tecnologías como flash o actionscript. Además HTML5 tiene soporte para audio y video. También cabe mencionar que, HTML5 define una sintaxis que es compatible con HTML4 y XHTML 1.0.
  11. thead (título), tbody (cuerpo de tabla), tr (renglón), td (contenido/ columna), tfoot (footer).
